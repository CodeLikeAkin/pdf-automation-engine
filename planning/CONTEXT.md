# Planning Context: Strategic Architecture

In this room, we define the roadmap for each project. We don't just "list chapters"; we build a logical progression of value.

## The Planning Workflow
1. **Define Objective**: What problem are we solving for the reader?
2. **Select Theme**: Choose from `vitality`, `vanguard_security`, or `premium`.
3. **Structure Outline**: 9-12 chapters, grouped into 3 thematic "Parts".
4. **Draft Specs**: Create a `.md` spec file with chapter objectives and key takeaways.

## Integration with Automation
- **Project Slugs**: Decide on the `project_slug` early (e.g., `the-citizens-shield`).
- **Asset Planning**: Identify if specific cover images or data visualizations are needed.
- **Design Tokens**: Define the primary/accent color strategy to be used in `MASTER.md`.
