# Skills Context: AI Persona Definitions

In this room, we define the "Brain" of the automation engine. We use persona-based skills to ensure consistent quality across different project phases.

## Core Personas
- **`co-authoring.md`**: The Strategic Drafter. Focuses on structure, authority, and depth.
- **`humanizer.md`**: The Editorial Polisher. Focuses on tone, flow, and removing "AI-isms".
- **`ui-ux-pro-max.md`**: The Visual Architect. Defines the standards for type scales, grids, and premium PDF elements.
- **`visual-audit.md`**: The Forensic Auditor. Used to identify design bugs and layout inconsistencies.

## Usage
- These files should be treated as "System Prompts" or "Instruction Sets" for the AI during specific tasks.
- When drafting, always reference the `co-authoring` and `humanizer` skills.
- When modifying the renderer, always adhere to the `ui-ux-pro-max` standards.
